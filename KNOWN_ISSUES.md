  <h1>已知问题与当前限制</h1>

  <h2>自动 AI 监视默认关闭</h2>
  <ul>
    <li>现象：全新安装后不会自动把首页推荐发给 AI。</li>
    <li>处理方式：在设置中勾选「启用自动 AI 监视」并确认风险提示，或点击右下角 ▶。</li>
    <li>状态：v0.7.3 起的默认隐私策略，不是故障。</li>
  </ul>

  <h2>网站改版可能暂时影响卡片识别</h2>
  <ul>
    <li>现象：B 站或 YouTube 修改首页结构后，新样式视频卡片可能暂时无法被脚本识别。</li>
    <li>安全行为：无法完整读取视频 ID、标题时保持显示，不调用 AI，也不自动写入黑名单；频道标识缺失时仍可右键手动隐藏当前视频。</li>
    <li>状态：属于外部网站结构变化风险；v0.7.2 已覆盖当前 B 站卡片，以及 YouTube 传统、现代、<code>-wiz</code> 实验卡片和 Shadow DOM 内缩略图/标题结构。</li>
  </ul>

  <h2>默认模型名称可能随服务商调整</h2>
  <ul>
    <li>现象：AI 服务商可能更名、停用模型，或让不同账号拥有不同的可用模型列表。</li>
    <li>处理方式：设置面板允许直接修改模型名；连接失败时应先按对应服务商控制台显示的模型名称修改并测试。</li>
    <li>状态：外部 API 变化风险，不影响本地规则、黑白名单和暂停功能。</li>
  </ul>

  <h2>正则表达式采用安全子集</h2>
  <ul>
    <li>现象：可能造成浏览器长时间回溯的复杂正则会被拒绝保存，即使它在 JavaScript 语法上有效。</li>
    <li>处理方式：优先使用普通关键词；复杂规则拆成多行简单关键词或带明确分隔符的正则。</li>
    <li>状态：v0.7.0 的主动安全限制，用于避免单条规则让首页卡死。</li>
  </ul>

  <h2>已修复问题</h2>
  <p>以下问题最初记录于 v0.1.0，已在 v0.2.0 中处理。</p>

  <h2>设置面板遮挡首页视频</h2>
  <ul>
    <li>现象：设置面板尺寸较大，展开后会遮挡首页右侧较多视频内容。</li>
    <li>处理方式：面板宽度缩小为 360 像素，API、黑名单和学习信息改为折叠区域，并限制列表高度。</li>
    <li>状态：已在 v0.2.0 修复。</li>
  </ul>

  <h2>浮动按钮与“返回顶部”按钮重叠</h2>
  <ul>
    <li>现象：页面右下角“AI 过滤”按钮可能遮挡 B 站原有的向上/返回顶部按钮。</li>
    <li>处理方式：桌面端浮动按钮向左移动到 B 站右侧工具栏外侧；窄屏下改为底部避让。</li>
    <li>状态：已在 v0.2.0 修复。</li>
  </ul>
