B站首页 AI UP 主过滤器

厌倦了 B 站首页反复出现的擦边、猎奇、标题党、焦虑营销或其他不想看的推荐？这个脚本允许你直接用一句自然语言描述偏好，再结合本地规则、手动“不喜欢”样本和 AI 语义判断，逐步整理出更清爽的首页。

脚本不会因为 AI 一次判断就永久拉黑整个 UP 主。AI 命中时只隐藏当前视频；同一 UP 主多次命中后，只会显示拉黑建议，最终决定权仍在用户手中。

主要功能

  • 使用自然语言描述不想看到的内容。
  • 首页首屏、无限滚动和“换一换”出现的新视频都可自动识别。
  • 本地标题关键词、正则规则、UP 主黑名单和白名单优先判断。
  • 只有本地规则和缓存无法确定时才调用 AI，减少 API token 消耗。
  • 在视频卡片上点击右键，可标记“不喜欢此视频”、手动拉黑 UP 主或将 UP 主加入白名单。
  • 手动标记后，AI 可以分析标题特征并更新本地偏好画像。
  • AI 可提炼候选标题规则，但必须由用户确认后才会启用。
  • 浮动暂停按钮可停止首页自动 AI 监视，本地规则和已有缓存仍继续生效。
  • 支持配置与黑白名单导入、导出；导出文件不包含 API Key 和 AI 判断缓存。
  • API 失败时保持视频显示，不会因为网络或模型故障误隐藏内容。

支持的 AI 服务商

当前支持 DeepSeek、AiHubMix、OpenAI、Google Gemini、Anthropic Claude、豆包（火山方舟）、阿里云百炼（通义千问）、智谱 GLM、Kimi（月之暗面）、腾讯混元和百度千帆（ERNIE）。

脚本会自动处理各平台的请求地址、鉴权头、输出 token 参数和响应格式。每个平台的模型名与 API Key 分开保存在当前 Tampermonkey 脚本的本地存储中，切换服务商不会覆盖其他平台配置。

如何使用

  1. 安装 Tampermonkey 和本脚本，打开或刷新 B 站首页。
  2. 点击右下角“AI 过滤”按钮。
  3. 填写要过滤的内容描述。
  4. 展开“API 与备份”，选择服务商，填写该平台的 API Key，并点击“测试连接”。
  5. 保存设置后，脚本会从本地规则和缓存开始处理首页推荐。
  6. 遇到特别不喜欢的视频时，在卡片上点击右键并选择“不喜欢此视频 · 隐藏并让 AI 学习”。

如何节省 token

  • 白名单、黑名单、标题关键词和正则全部在浏览器本地运行。
  • 匹配与不匹配结果都会缓存，视频重新出现时不重复询问 AI。
  • 一次最多批量判断 10 个视频，减少重复提示词。
  • 限制标题、UP 主名、偏好画像和历史样本的发送长度。
  • 输出额度按视频数量动态计算，只有 JSON 为空或被截断时才提高额度补救一次。
  • 对支持的平台默认关闭不必要的深度思考或联网增强。
  • 可以随时暂停首页自动 AI 监视；暂停期间手动右键添加的不喜欢样本仍可按用户明确操作进行学习。

隐私与安全

只有需要 AI 判断或学习时，脚本才会把过滤描述、偏好画像、近期不喜欢样本的视频标题、待判断视频标题和页面显示的 UP 主名称发送给你选择的 API 服务商。脚本不会主动发送 B 站 Cookie、账号资料或页面正文。

API Key 仅保存在当前 Tampermonkey 脚本的本地存储中，不会写入导出文件。请注意，浏览器本地存储不等同于系统级密码保险箱，请使用单独创建、额度受控的 API Key。

当前范围

当前版本只处理 https://www.bilibili.com/ 首页，不处理搜索页、排行榜、视频页、B站 App、观看历史或自动云同步。按住 Shift 再点击右键，可以使用浏览器原生右键菜单。
