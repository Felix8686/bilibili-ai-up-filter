B站首页 AI UP 主过滤器

当前版本：v0.4.2

作者：Felix8686

这是一个 Tampermonkey 用户脚本。它会先用本地白名单、黑名单、标题关键词和正则规则判断 B 站首页推荐；只有本地规则无法确定且没有可用缓存的视频，才会批量交给用户配置的 AI 接口进行语义判断。AI 命中只隐藏当前视频，不会自动永久拉黑整个 UP 主；同一 UP 主多次高置信度命中后，脚本会提示用户自行确认是否拉黑。

当前版本范围

  • 只处理 https://www.bilibili.com/ 首页。
  • 支持首页首屏、无限滚动和“换一换”产生的新卡片。
  • 支持在首页视频卡片上右键添加不喜欢样本，并自动生成 AI 偏好画像；兼容 BV、AV 链接和特殊悬停预览卡片。
  • 支持右键手动拉黑 UP 主或加入 UP 主白名单。
  • 支持本地标题黑白名单，规则可以是普通关键词或 /正则表达式/。
  • 不处理搜索页、排行榜、视频页、B站 App 或观看历史。
  • 判断失败时保持视频显示，不会因为 API 故障自动拉黑。

安装

  1. 在 Chrome 或 Edge 安装 Tampermonkey。
  2. 在 Tampermonkey 中新建脚本。
  3. 复制 bilibili-ai-up-filter.user.js 的全部内容并保存。
  4. 打开或刷新 B 站首页，右下角会显示“AI 过滤”按钮。

配置

  1. 点击页面右下角“AI 过滤”，或使用 Tampermonkey 菜单中的“打开 B站首页 AI 过滤设置”。
  2. 填写要过滤的内容描述，例如“擦边、夸张猎奇、标题党的内容”。
  3. 选择 DeepSeek 或 AiHubMix，填写模型名和对应的 API Key。
  4. 点击“测试连接”，确认接口可以返回脚本要求的 JSON。
  5. 按需填写本地标题黑名单和白名单，每行一条关键词或 /正则表达式/。
  6. 点击“保存”，脚本会开始判断首页推荐。

DeepSeek 默认模型为 deepseek-v4-flash，AiHubMix 默认模型为 gpt-4o-mini。如果服务商调整模型名称，可以直接在设置中修改。

本地规则与 AI 判断顺序

  1. 手动标记过“不喜欢”的视频 ID（BV 或 AV）：隐藏当前视频。
  2. UP 主白名单或标题白名单命中：保持显示，不再调用 AI。
  3. UP 主黑名单或标题黑名单命中：直接隐藏，不再调用 AI。
  4. 以上规则都无法确定时：读取当前过滤条件对应的 AI 判断缓存。
  5. 仍无结果时：把视频加入批量 AI 判断队列。

白名单会覆盖 AI 判断和普通黑名单规则。唯一例外是用户明确对某个具体视频执行了“不喜欢此视频”，该视频仍会隐藏。标题规则支持普通包含匹配，也支持用一对斜杠包围的正则表达式，例如 /月入.*万/。

右键菜单与主动学习

  1. 在 B 站首页的视频卡片上点击右键。
  2. 选择“不喜欢此视频 · 隐藏并让 AI 学习”，只处理当前视频 ID。
  3. 也可以选择“手动拉黑该 UP 主”，隐藏该 UP 主之后的首页推荐。
  4. 对希望始终显示的作者选择“始终显示该 UP 主”，加入 UP 主白名单。
  5. 该视频的 BV 或 AV 号会立即进入本地不喜欢样本并从首页隐藏。
  6. 配置了 API Key 时，AI 会自动分析标题和 UP 主名称，提炼内容特征并更新偏好画像。
  7. 后续首页推荐会同时参考手写过滤描述、偏好画像和近期不喜欢样本。

AI 高置信度命中也只隐藏当前视频。同一 UP 主累计至少 3 个不同视频命中后，设置面板会显示“拉黑”或“忽略”建议，只有用户点击“拉黑”才会写入永久 UP 主黑名单。按住 Shift 再点击右键，可以使用浏览器原生右键菜单。未配置 API Key 时样本仍会保存，保存有效配置后会自动补充分析。

AI 提炼候选规则

点击设置面板中的“AI 提炼候选规则”，AI 会根据过滤描述、学习画像和最近样本建议可在本地运行的标题关键词。候选规则不会自动启用，用户必须逐条点击“采用”。采用后，后续命中由浏览器本地完成，不再为这些内容消耗 AI token。

暂停自动 AI 监视

  • 浮动“AI 过滤”按钮右侧的 ⏸ 用于暂停自动 AI；暂停后按钮变为 ▶，点击即可恢复。
  • 暂停状态会保存在 Tampermonkey 本地存储中，刷新页面后仍然保持。
  • 暂停期间，本地标题规则、UP 主黑白名单、视频不喜欢样本和已有 AI 缓存继续生效；页面滚动和新推荐不会自动产生 API token 消耗。
  • 暂停期间右键标记“不喜欢”属于用户明确触发的操作：配置了 API Key 时仍会立即隐藏、分析并学习，但不会恢复首页自动监视。
  • 暂停前积压且未由用户再次操作的待学习样本会继续等待，恢复后才自动批量处理。
  • “测试连接”和“AI 提炼候选规则”属于用户主动操作，即使暂停自动监视，也仍可手动调用。
  • 点击暂停前已经发送的单次请求可能已经产生 token；脚本会忽略它稍后返回的旧结果，并阻止后续自动队列。

Token 节省机制

  • 本地优先：白名单、UP 主黑名单、视频不喜欢样本、标题关键词和正则规则都在浏览器内判断，命中时完全不调用 AI。
  • 持久缓存：AI 对一个视频 ID（BV 或 AV）的匹配和不匹配结果都会缓存在 Tampermonkey 本地存储中。刷新页面、无限滚动或卡片重新出现时可直接复用。
  • 自动失效：缓存同时绑定 API 服务商、模型、过滤描述、学习画像及最近样本；这些语义条件变化后不会误用旧判断。缓存最多保留 600 条，当前版本不启用按 30 天过期。
  • 只发模糊内容：只有本地规则无法确定且没有有效缓存的视频才会进入 AI 队列。
  • 随时暂停：使用浮动暂停按钮可停止首页自动判断和待学习样本的自动分析，本地规则与缓存仍正常工作。
  • 批量请求：一次最多判断 10 个视频，减少重复系统提示和网络请求开销。
  • 缩短输入：只发送最近 3 个学习样本；标题最多 160 字、UP 主名称最多 60 字，过滤描述和学习画像各最多 400 字；历史样本不再重复发送 UP 主名称。
  • 压缩输出：不匹配项的原因要求返回空字符串，首次请求的输出 token 上限会按本批视频数量动态计算。
  • 异常时才扩容：首页分类、主动学习和候选规则生成共用同一补救流程；如果首次返回空内容、截断 JSON 或缺少必要字段，只提高输出额度重试一次。补救请求最低 1200、最高 2400 token，正常请求仍保持原来的低额度。
  • 避免无效重试：JSON 格式错误在内部补救失败后不再进入外层同参数重复队列；网络错误、限流和服务端故障仍按原策略重试。
  • 可见统计：面板状态会显示本页由本地规则和缓存省下的 AI 判断次数，以及实际发送给 AI 的视频数量。

黑名单与备份

  • 黑名单使用 UP 主 UID 作为唯一键，保存在 Tampermonkey 本地存储中。
  • UP 主白名单同样按 UID 保存，并优先于自动判断和普通黑名单。
  • 手动不喜欢样本使用规范化的视频 ID（BV 或 AV）作为唯一键，与 UP 主黑名单分开保存。
  • 可在设置面板中删除误判的黑名单、白名单和未采用的 AI 候选规则。
  • “导出”会保存过滤描述、非敏感设置、UP 主黑白名单、标题黑白名单、不喜欢样本和学习画像。
  • 导出文件不会包含 API Key 或 AI 判断缓存。
  • “导入”会按 UID 和视频 ID 合并数据，并保留当前设备的 API Key；v0.1.0、v0.2.0 和 v0.3.x 备份均可导入。

隐私说明

只有本地规则和缓存无法判断时，AI 判断才会把过滤描述、偏好画像、近期不喜欢样本的视频标题，以及待判断视频的标题和页面显示的 UP 主名称发送给所选 API 服务商。主动学习会发送当前手动不喜欢样本的标题和 UP 主名称。脚本不会主动发送 B 站 Cookie、账号资料或页面正文。API Key 仅存放在当前 Tampermonkey 脚本存储中，但浏览器本地存储不等同于系统级密码保险箱。
