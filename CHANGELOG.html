版本记录

v0.4.2（2026-07-15）

增强首页特殊推荐卡片的视频标识与右键事件识别，修复个别视频无法弹出屏蔽菜单的问题。

  • 视频标识解析从仅支持 BV... 扩展为同时支持 BV... 和 av...。
  • 支持从卡片的 data-bvid、data-aid、data-av 以及编码后的跳转地址中提取视频 ID。
  • 右键菜单不再只检查事件的第一个目标，而会沿完整事件路径寻找所属视频卡片。
  • 增加页面坐标下层元素兜底，兼容悬停播放器、覆盖层和部分特殊卡片结构。
  • 旧版 BV 学习样本、判断缓存和备份继续兼容，不需要用户迁移数据。
  • 核心自动化测试增加到 21 项；首页集成测试新增“AV 链接 + Shadow DOM 悬停覆盖层”回归场景。

v0.4.1（2026-07-15）

把“暂停自动 AI”进一步限定为暂停首页后台监视，不阻止用户明确触发的手动学习。

  • 暂停状态下右键添加“不喜欢”时，视频仍立即隐藏并调用 AI 分析当前样本。
  • 手动学习请求不受监视运行代次影响；即使学习期间切换暂停状态，也会保存这次用户明确请求的结果。
  • 手动学习完成后不会自动恢复首页监视，也不会在暂停期间顺带处理其他历史积压样本。
  • 没有 API Key 时样本仍仅保存在本地，恢复自动 AI 后再处理。
  • 首页集成测试新增“暂停监视、手动学习成功、首页请求数不增加、状态仍暂停”的回归场景。

v0.4.0（2026-07-15）

新增独立的自动 AI 暂停开关，让用户在保留本地过滤能力的同时控制 token 消耗。

  • 浮动“AI 过滤”按钮右侧新增 ⏸ 按钮；暂停后显示 ▶ 和“AI 已暂停”状态。
  • 暂停状态持久保存，刷新页面或重新打开浏览器后不会自行恢复。
  • 暂停期间停止新的首页 AI 分类请求，但本地标题规则、UP/BVID 黑白名单和已有缓存继续生效。
  • 暂停期间右键不喜欢样本只保存和隐藏，不自动调用 AI；恢复后再处理待学习样本。
  • 连接测试和 AI 候选规则生成属于用户主动操作，不受自动监视暂停影响。
  • 使用运行代次隔离暂停前已经发出的请求；旧响应不再写入判断结果、缓存或偏好画像。
  • 备份中的非敏感设置会包含暂停状态，旧备份缺少该字段时默认恢复为未暂停。
  • 核心自动化测试保持 20 项全通过；首页集成测试新增启动暂停、本地规则继续、暂停学习和恢复处理场景。

v0.3.2（2026-07-15）

修复主动学习结果格式错误，并把结构化 JSON 补救统一到全部 AI 功能。

  • 首页视频分类、右键不喜欢主动学习和 AI 候选规则生成现在共用一次自适应 JSON 补救流程。
  • 主动学习首次输出为空、截断或缺少字段时，会把 550 token 临时提高到最多 1650 token 重试一次。
  • 候选规则首次格式异常时，会把 320 token 临时提高到 1200 token 重试一次。
  • 学习结果兼容 summary、features、learned_profile 和 profile 等安全字段别名。
  • 候选规则兼容 titleBlacklist、titleWhitelist、blacklistRules 和 whitelistRules。
  • 学习和候选规则状态提示会明确标出首次 JSON 异常是否已自动修复。
  • 作者署名从 local 更新为 Felix8686。
  • 核心自动化测试增加到 20 项；首页集成测试新增主动学习空输出、一次扩容及字段别名回归场景。

v0.3.1（2026-07-14）

修复结构化 JSON 输出偶发为空或被截断的问题，同时避免无效重试造成额外 token 消耗。

  • 首次分类请求继续使用按视频数量动态计算的低输出额度。
  • 检测到空内容、截断 JSON、缺少结果或其他可解析格式错误时，只进行一次自适应补救请求。
  • 补救请求强化“立即输出完整 JSON”指令，并把输出额度提高到 1200～2400 token。
  • 读取 finish_reason、完成 token 和推理 token 信息，用于区分空输出与 finish_reason=length 截断。
  • 补救失败后不再把格式错误放回同参数重试队列；网络错误、限流和服务端故障仍保留原重试策略。
  • 连接测试和批量判断会提示是否使用过自动修复。
  • 核心自动化测试增加到 18 项；首页集成测试新增“首次截断、一次扩容恢复”回归场景。

v0.3.0（2026-07-14）

重构为本地规则优先、AI 只处理模糊内容的混合过滤流程，并降低误拉黑风险和 token 消耗。

  • AI 高置信度命中只隐藏当前视频，不再自动把整个 UP 主加入永久黑名单。
  • 同一 UP 主累计至少 3 个不同视频命中后显示建议，只有用户确认才拉黑，也可忽略本轮建议。
  • 右键菜单新增“手动拉黑该 UP 主”和“始终显示该 UP 主”。
  • 新增 UP 主白名单；白名单优先于 AI 判断和普通黑名单。
  • 新增本地标题黑白名单，支持普通关键词和 /正则表达式/。
  • 新增手动触发的 AI 候选规则提炼；建议不会自动启用，必须由用户逐条采用。
  • 本地规则命中时不调用 AI；只有无法本地判断且没有缓存的视频才进入 AI 队列。
  • 新增最多 600 条的持久 AI 判断缓存，并用服务商、模型、过滤描述和学习画像生成条件键，避免条件改变后误用旧结果。
  • 一次最多批量判断 10 个视频；近期学习样本由 8 条缩减为 3 条。
  • 限制标题、UP 主名称、过滤描述和学习画像的发送长度，历史样本不再重复发送 UP 主名称。
  • 不匹配结果不再要求 AI 输出原因，输出 token 上限改为按批量大小动态计算。
  • 状态栏新增本地规则命中、缓存命中和实际送 AI 数量统计。
  • 备份格式升级到 schema v3，加入 UP 主白名单和标题规则；API Key 与 AI 判断缓存仍不导出，并兼容 v1、v2 备份。
  • 核心自动化测试增加到 15 项，并扩展首页集成测试覆盖本地规则、白名单、缓存复用和用户确认拉黑。

v0.2.0（2026-07-14）

加入手动不喜欢与 AI 主动学习，并完成首批界面修复。

  • 新增首页视频卡片自定义右键菜单，可把当前 BVID 加入不喜欢样本并立即隐藏。
  • 手动样本与 UP 主黑名单分开保存，不会因一次手动标记误拉黑整个 UP 主。
  • AI 会自动分析不喜欢样本，提炼内容特征并持续更新偏好画像。
  • 后续首页判断会综合过滤描述、偏好画像及近期手动样本。
  • 没有 API Key 时保留待分析样本，保存有效配置后自动补学。
  • 设置面板改为 360 像素紧凑折叠布局，并限制列表区域高度。
  • 浮动按钮移至页面右侧工具栏左侧，避免与返回顶部按钮重叠。
  • 导入导出加入不喜欢样本和偏好画像，并兼容 v0.1.0 备份。
  • 核心自动化测试由 8 项增加到 11 项，并扩展首页集成测试。

v0.1.0（2026-07-14）

首个可运行版本。

  • 识别 B 站首页首屏和动态加载的视频推荐卡片。
  • 支持 DeepSeek 与 AiHubMix 的批量 AI 语义判断。
  • AI 命中后按 UP 主 UID 写入本地黑名单并隐藏相关推荐。
  • 提供基础配置、连接测试、黑名单删除及 JSON 导入导出。
  • API 故障时保持视频显示，避免故障造成误隐藏。
